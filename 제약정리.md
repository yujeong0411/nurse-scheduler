## 현재 시스템 전체 정리 (정확한 버전)

### 1. 신청 코드 종류
| 분류 | 코드 |
| :--- | :--- |
| **근무** | D, E, N, D9, D1, 중1, 중2 |
| **휴무** | OFF, 주, 법휴, 수면, 생휴, 휴가, 특휴, 공가, 경가, 보수, 병가, POFF, 필수, 번표 |
| **제외** | D 제외, E 제외, N 제외 |
| **OR** | `/` 구분자 (예: D/휴가) |

---

### 2. Hard / Soft 구분

#### **항상 Hard (H8에서 강제 배정)**
| 코드 | 이유 |
| :--- | :--- |
| **병가** | 병가 기간 반드시 휴무 |
| **번표** | 번표 교환은 반드시 배정 |
| **수면** | 명시 신청 시 이월 수면 사용 보장 |
| **D9, D1** | 입력전용 — hard 아니면 자동배정 불가 |

#### **항상 Hard (H9에서 강제 금지)**
* **코드:** D 제외, E 제외, N 제외

#### **나머지 Soft** (번표·수면 명시신청 제외)
* A/B 조건 + score로 우선순위 결정
* **특휴·공가·경가·보수·필수:** soft이지만 신청하지 않은 날에는 자동 배정 불가 (요청일에만 배정 가능)

#### **OFF 신청 vs 구체적 휴무 신청**
| 신청 코드 | 배정 충족 조건 |
| :--- | :--- |
| **OFF** | 어떤 휴무 타입이든 배정되면 충족 (생휴·수면·주 등 포함) |
| **휴가·수면·생휴·특휴 등** | 정확히 그 타입으로 배정되어야 충족 |

> OFF는 "아무 날이나 쉬고 싶다"는 의미이므로 포괄적으로 처리.  
> 자동배정 휴무(생휴·수면)는 OFF 신청한 날에 얹혀서 배정될 수 있음.

### 3. A/B 조건 + Score

* **Score**
    * 초기값: 100점 (매월 리셋)
    * A 신청 1개: -1점
    * B 신청 1개: -3점
* **A조건 제한:** 월 최대 3개
* **S1 가중치**
    * **A:** $800 + \text{score} \times 5$
    * **B:** $250 + \text{score} \times 5$

| score | A weight | B weight |
| :--- | :--- | :--- |
| 100 | 1300 | 750 |
| 85 | 1225 | 675 |
| 70 | 1150 | 600 |
| 0 | 800 | 250 |

* **특이사항**
    * A weight > B weight 항상 보장 (A 최저 800 > B 최고 750)
    * A 최저 (score 0) = 800, B 최고 (score 100) = 750
    * score 1점 차이 → weight 5 차이 → 공정성 페널티 최대 ~25 수준이므로 역전 어려움
    * **같은 날 여러 신청 (D + E + 중1):** 각각 S1에 별도 항 → H1(하루 1배정)에 의해 자동으로 OR 동작
    * **OR 신청 (D/휴가):** S1a에서 그룹 처리 (하나 배정 시 보상)

### 4. Hard Constraints

| 번호 | 내용 |
| :--- | :--- |
| **H1** | 하루 1배정 |
| **H2** | 일일 D/E/N 인원 정확히 고정 |
| **H2a** | 중2 role 아닌 간호사 → 중2 불가 |
| **H2b** | D9/D1/중1 → hard 요청 없으면 자동배정 불가 |
| **H3** | 역순 금지 (D→중2→E→N 순서만) |
| **H3a** | N→1휴무→D·중간·N 금지 (E만 허용) / N→N 연속(NNN)은 허용 |
| **H3c** | N 다음날 보수/필수/번표 금지 |
| **H4** | 최대 연속 근무 5일 |
| **H5** | 최대 연속 N 3개 |
| **H6** | NN 블록 종료 후 2일 휴무 |
| **H8** | 병가/D9/D1 hard 배정 |
| **H9** | 제외 요청 강제 금지 |
| **H10** | 고정 주휴요일에 주 배정 |
| **H10a** | 주는 고정 주휴일에만 배치 가능 |
| **H10b** | 법휴는 공휴일에만 배치 가능 |
| **H11** | 주당 OFF 정확히 1개 (주4일제 2개) |
| **H12** | 매 근무 책임 1명 이상 |
| **H13** | 매 근무 책임+서브차지 N명 이상 |
| **H14** | 역할 누적 인원 제한 |
| **H15** | 책임만 1명 이하 |
| **H17** | 임산부 4연속 근무 금지 |
| **H18** | 공휴일 비근무 시 법휴만 허용 |
| **H19** | 임산부 4연속 근무 후 POFF 강제 |
| **H20** | 휴무 편차 ±2 이내 (주4일제 +4) |
| **H21** | 신청 휴무 샌드위치 금지 — 간호사가 d일에 휴무 신청 시, 양쪽(d-1·d+1)이 모두 휴무 배정이면 d일도 반드시 휴무 |
| **특수OFF** | 생휴(여성 월 1회 자동·신청 시 해당일)/수면(자동 또는 명시 신청 시 hard)/번표·병가(hard, 요청 수 == 배정)/특휴·공가·경가·보수·필수(soft, 신청일에만 배정 가능)/휴가(catch-all >=) |

### 5. Soft Constraints (목적함수)

| 번호 | 내용 | 가중치 |
| :--- | :--- | :--- |
| **S1** | 희망 요청 반영 | A: +800+score×5 / B: +250+score×5 |
| **S1a** | OR 신청 그룹 | 동일 공식 |
| **S1b** | hard 휴무 사이 빈날 OFF 유도 | +250 |
| **S2** | D/E/N 횟수 균등 | -5 × (max−min) |
| **S3** | N 균등 분배 | -8 × (max−min) |
| **S4** | 주말 근무 균등 | -8 × (max−min) |
| **S5** | 일반(비책임) 초과 권고 | -3 × 초과수 |
| **S6** | N 연속 배정 보상 | +20 / NN쌍 |
| **S7** | 연속 휴무 보상 | +15 / 연속휴무쌍 |
| **S8** | 월 N 초과 억제 | -300 / 초과 N 1개 |

* **참고:** S8은 월 N 6개 권고를 소프트로 적용. 인원 부족 시 초과 허용하되 강하게 억제.

---

### 6. 전체 우선순위 구조

1.  **병가 / 번표 / 수면(명시 신청) / D9 / D1** → hard (반드시 배정)
2.  **제외 요청** → hard (반드시 금지)
3.  **A조건 신청** → soft-high (800 + score×5)
4.  **B조건 신청** → soft (250 + score×5)
5.  **공정성 균등** → 페널티 (S2~S4)
6.  **연속 패턴 보상** → 보너스 (S6, S7)
7.  **월 N 초과 억제** → 페널티 (S8, -300/초과 N)
